Deployment-Wege im Überblick

📤 Export als Package (.zip)

Im Maker Portal: App → Export package. Die .zip-Datei enthält die App und Verbindungsreferenzen.

  • Import in andere Umgebung über „Import canvas app"
  • Kein Dataverse oder Solution nötig
  • Wahl: als neue App erstellen oder bestehende aktualisieren
✓ Einfach & schnell ✗ Kein ALM / Versionierung

💾 Save As .msapp (lokal)

Speichert die App-Datei direkt auf den eigenen Rechner. Gut als persönliches Backup.

  • Power Apps Studio → Save As → This computer
  • Kann später wieder geöffnet werden
  • Kein Deployment-Mechanismus!
✓ Backup ✗ Nur manuell ✗ Kein Sharing

🔗 Direkt teilen (Share)

App veröffentlichen → im Maker Portal „Share" wählen → Benutzer oder Gruppen hinzufügen.

  • Rolle wählen: User (nutzen) oder Co-owner (bearbeiten)
  • Optionale E-Mail-Einladung senden
  • Kein „Transport" — App bleibt in der gleichen Umgebung
✓ Sofort verfügbar ℹ Gleiche Umgebung

📦 Dataverse Solution (empfohlen)

Canvas App einer Solution hinzufügen → Export als Managed oder Unmanaged Solution → Import in Zielumgebung.

  • ALM mit DevOps / GitHub Actions möglich
  • Alles in einem Paket: App + Tabellen + Flows + Rollen
  • Versionierung, Dependency Tracking, Layers
✓ Best Practice ✓ Professionelles ALM ⚠ Braucht Dataverse

💡 Wichtig: Speichern ≠ Veröffentlichen

Wenn du speicherst, sehen deine Benutzer noch die alte Version. Erst nach „Publish" werden die Änderungen live. Denk daran wie einen Draft vs. eine Veröffentlichung.

3

Umgebungen (Environments)

Umgebungstyp Dataverse Zweck Wer hat Zugriff?
Default ✓ Wird automatisch erstellt. Jeder im Tenant kann hier arbeiten. Alle Benutzer
Developer ✓ Persönliche Entwicklungsumgebung zum Lernen und Testen. Nur der Ersteller
Sandbox ✓ Test / UAT. Kann zurückgesetzt (Reset) werden. Zugewiesene User
Production ✓ Produktiver Betrieb. Kein Reset möglich. Zugewiesene User
Trial ✓ Testumgebung, wird nach 30 Tagen gelöscht. Ersteller + eingeladene

⚠ Achtung: Die Default-Umgebung

Die Default-Umgebung ist keine Spielwiese — sie ist für alle sichtbar und hat keinen Schutz. Für ernsthafte Entwicklung immer eine eigene Dev-Umgebung erstellen! Admins können die Erstellung von Umgebungen im Power Platform Admin Center einschränken.

5

Schnell-Referenz: Versionen verwalten

🔄 App-Versionen im Maker Portal

Unter Apps → ··· → Details → Versions findest du alle gespeicherten Versionen deiner App.

  • Die zuletzt veröffentlichte Version ist als „Live" markiert
  • Du kannst Versionen der letzten 6 Monate wiederherstellen
  • Restore erstellt eine neue Version (die alte bleibt erhalten)
  • Du kannst nur die höchste Version veröffentlichen
  • Es gibt keinen „Undo Publish" — immer Restore → neue Version → Publish
